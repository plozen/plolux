# KCL 결제 시스템 구축 및 운영 가이드 (Payment Strategy)

이 문서는 KCL 프로젝트의 Phase 2 수익화 단계에서 필요한 결제 시스템 도입 전략, 행정 절차, 그리고 기술적 준비 사항을 정리합니다.

---

## 1. 결제 도입을 위한 행정 절차

실제 대금 정산(Settlement)을 받기 위해서는 법적 지위(사업자)가 반드시 필요합니다.

### 1.1. 사업자 등록 가이드

- **단계**: 개발 초기(테스트 모드)에는 필요 없으나, 실결제 오픈 2~4주 전에는 완료되어야 함.
- **추천 유형**: **개인사업자 - 간이과세자**
  - 연 매출 8,000만 원 미만 시 부가세 혜택이 큼.
  - 홈택스(Hometax)를 통해 온라인으로 10분 내 신청 가능.
- **필수 업종**: 전자상거래업 (통신판매업)

### 1.2. 필수 서류 목록

PG사(결제 대행업체) 심사를 위해 다음 서류가 준비되어야 합니다.

1. 사업자등록증 사본
2. 통신판매업 신고증 (정부24에서 신청)
3. 대표자 명의 통장 사본 (정산 대금 입금용)
4. 대표자 신분증 사본

---

## 2. 기술적 통합 전략 (Technical Integration)

### 2.1. 결제 솔루션 선택: PortOne (포트원, 구 아임포트)

- **이유**: 하나의 SDK로 카카오페이, 네이버페이, 신용카드, PayPal을 통합 관리 가능.
- **구조**:
  - **Frontend**: PortOne SDK를 통한 결제창 호출.
  - **Backend (Supabase)**: Edge Functions를 이용한 결제 검증(Verification) 및 Webhook 처리.

### 2.2. 개발 단계 (Sandbox vs Production)

1. **테스트 단계 (현재 ~ Phase 2 개발기)**:
   - 사업자 없이 '테스트 상점 아이디' 발급.
   - 가상 결제를 통해 결제 성공/실패 로직 완벽 구현.
2. **운영 단계 (Phase 2 런칭 시점)**:
   - 사업자 등록 후 PG사 심사(약 1~2주 소요).
   - 실상점 ID 및 API Key로 교체.

## 3. 수익 모델별 정산 구조

| 모델                  | 정산 주기           | 수수료 (보수적 예측) | 비고              |
| --------------------- | ------------------- | -------------------- | ----------------- |
| **신용카드/간편결제** | 결제일 + 5~7 영업일 | 약 3.0% ~ 3.5%       | 국내 표준         |
| **PayPal (해외)**     | 결제일 + 3~5 영업일 | 약 4.4% + $0.30      | 글로벌 팬덤용     |
| **광고 (AdSense)**    | 익월 21~26일        | -                    | $100 이상 시 지급 |

---

## 4. 보안 및 정책 (Security & Policy)

- **결제 검증**: 클라이언트에서 넘어온 결제 완료 응답을 믿지 않고, 반드시 서버(Supabase)에서 PortOne API를 통해 실제 입금 여부를 2차 검증(Cross-check).
- **환불 정책**: 디지털 아이템(부스터, 커피차 등) 특성상 사용 즉시 효력이 발생하므로 '청약철회 제한' 고지 필수.
- **개인정보**: 결제 시 수집되는 개인정보는 PG사가 관리하며, 서비스 DB에는 최소한의 결제 식별값만 저장.

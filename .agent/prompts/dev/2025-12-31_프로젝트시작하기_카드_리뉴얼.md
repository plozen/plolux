# Task

현재 프로젝트의 `QuoteForm.tsx` 컴포넌트를 제공된 **새로운 기획 요구사항**에 맞춰 전면 리팩토링해주세요.
기존의 Step별 네비게이션 구조(`framer-motion` 활용)는 유지하되, `questions` 배열과 내부 렌더링 로직을 아래 명세대로 수정해야 합니다.

# Goal

사용자로부터 웹사이트 제작에 필요한 구체적인 정보를 6단계에 걸쳐 수집하고, 최종적으로 Supabase DB에 저장할 수 있는 형태여야 합니다.

# Step-by-Step Requirements (기획 상세)

### Step 1: 서비스 유형 선택 (Card 1)

- **질문(Title)**: "어떤 종류의 서비스가 필요하신가요?"
- **부제(Subtitle)**: "제작 목적에 맞는 구조를 선택해주세요."
- **유형**: Single Select (Card UI)
- **옵션**:
  1. **회사/브랜드 소개 홈페이지** (설명: 기업, 병원 등 신뢰도 목적 / Multi-page)
  2. **이벤트/광고용 랜딩 페이지** (설명: 상품 홍보, 신청 접수 / One-page)

### Step 2: 예산 범위 (Modified)

- **질문**: "예산 범위는 어떻게 되시나요?"
- **유형**: Single Select + '기타' 직접 입력
- **옵션**:
  - 35만원
  - 100만원
  - 200만원
  - 300만원
  - 미정
  - **기타**: 선택 시 `input` 필드가 나타나 직접 금액 입력 가능해야 함

### Step 3: 디자인 분위기 (Card 2)

- **질문**: "고객에게 어떤 느낌을 주고 싶으신가요?"
- **유형**: Single Select + 조건부 입력
- **옵션**:
  1. 신뢰감 있는 (Blue/Grey)
  2. 따뜻한/감성적인 (Beige/Pastel)
  3. 고급스러운 (Black/Gold)
  4. 개성 있는 (Vivid/Creative)
  5. **잘 모르겠어요** (기타)
     - _Action_: 이 옵션 선택 시, 참고할 사이트 URL이나 이미지를 첨부할 수 있는 `input` 또는 `file upload` UI 활성화

### Step 4: 기능 및 콘텐츠 (Card 3)

- **질문**: "사이트에 어떤 내용을 담고 싶으신가요?"
- **유형**: **Multi Select (다중 선택 가능)**
- **옵션 리스트**:
  1. 회사 소개 (연혁, 비전 등)
  2. 서비스 소개 (_선택 시: 서비스 개수 입력 필드 노출_)
  3. 자유게시판
  4. 포트폴리오
  5. SNS 홍보 채널 연동
  6. 다국어 지원
  7. 문의하기 (_선택 시: 수신 이메일/번호 입력 필드 노출_)
  8. 기타 (_선택 시: 텍스트 입력 필드 노출_)

### Step 5: 타겟 고객 (Card 4)

- **질문**: "주로 어떤 분들이 방문하길 원하시나요?"
- **유형**: Textarea (서술형)
- **가이드(Placeholder)**: "예: 2030 직장인 여성, 주말에 힐링하러 오는 분들 / 꼼꼼하게 학원 정보를 찾는 학부모 등"

### Step 6: 제작 목표 및 담당자 정보 (Card 5 + Existing ID 3)

- **구성**: 두 개의 섹션을 한 페이지에 표시
- **섹션 A (제작 목표)**:
  - 질문: "가장 얻고 싶은 핵심 목표는 무엇인가요?" (Single Select)
  - 옵션: 브랜드 신뢰도 상승, 문의/상담 예약, 상품 판매, 정보 전달
- **섹션 B (담당자 정보)**:
  - 기존 `QuoteForm`의 `id: 3` (multi-input) 내용을 그대로 가져옴
  - 필드: 담당자 성함, 연락처, 이메일, 기업명, (추가) 자료 첨부/설명

# Technical Constraints & Code Structure

1. **Data Structure Update**: `Question` 및 `InputField` 인터페이스를 확장하여 `isMultiSelect`, `hasEtcInput`, `conditionalInputs` 등의 속성을 처리할 수 있게 하세요.
2. **State Management**: `answers` 상태 객체가 위 6단계의 복합적인 데이터(단일값, 배열, 객체 등)를 모두 수용할 수 있어야 합니다.
3. **UI/UX**:
   - Step 3, 4의 조건부 입력창은 해당 옵션이 선택되었을 때만 부드럽게 나타나야 합니다 (`AnimatePresence` 활용).
   - Step 6은 내용이 많으므로 스크롤이 필요할 수 있음을 감안해주세요.
4. **Validation**: 각 단계의 필수 값(Required)이 채워지지 않으면 '다음' 버튼이 비활성화되어야 합니다.

위 명세를 바탕으로 `QuoteForm.tsx`와 관련 타입 정의를 작성해주세요.

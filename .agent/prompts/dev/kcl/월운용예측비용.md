# Cost Simulation (월 운영 비용 예측)

가장 중요한 것은 **"트래픽이 터졌을 때 서버비가 수익을 초과하지 않는 것"**입니다.

가정: DAU(일간 접속자) 10,000명 / 일일 총 클릭 수 1,000만 회 (헤비 유저 가정)

### 1. 고정 비용 (Infrastructure)

대부분의 PLOLUX 스택(Next.js, Cloudflare)은 트래픽 기반 과금 혹은 무료입니다.

| **항목** | **서비스** | **예상 비용 (월)** | **비고** |
| --- | --- | --- | --- |
| **Hosting** | Cloudflare Pages | **$0 (Free)** | 대역폭 무제한. 정적 자산 배포 최적2222. |
| **Database** | Supabase Pro | **$25** | 무료 티어는 동시 접속 폭주 시 멈출 수 있음. Pro 권장. |
| **Auth** | Supabase Auth | **$0** | MAU 50,000명까지 무료. |
| **Domain** | Namecheap/Gabia | **$1** | 연 $12 수준. |
| **합계** |  | **약 $26 (3.5만 원)** | 기본 인프라 유지비. |

### 2. 변동 비용 (Traffic-Driven) - 여기가 핵심

클릭커 게임은 **DB 쓰기(Write)**와 **Realtime 전송** 비용이 폭탄이 될 수 있습니다.

**A. 최악의 시나리오 (최적화 안 함)**

- 클릭 1회 = DB Write 1회 요청 시
- 1,000만 클릭 = 1,000만 번의 Edge Function 호출 + DB 쓰기.
- **예상 비용**: **월 $500 이상** 깨지거나 서비스 정지 당함. (절대 금지)

**B. PLOLUX 최적화 시나리오 (Redis Batching 적용)**

- **전략**: 유저의 클릭을 메모리(Redis)에 모았다가 **10초에 1번만 DB에 저장**.
- **Redis (Upstash)**: 1,000만 클릭을 처리해도 실제 커맨드는 줄어듦. (약 월 $10~$20 수준의 유료 플랜).
- **Edge Function**: 10초마다 호출되므로 호출 횟수 극감. (무료 범위 내 가능).
- **Realtime**: 접속자 수에 따라 과금되지만 Supabase Pro ($25) 내에 쿼터가 넉넉함. 추가 시 $10 당 1,000 동시 접속.

### 최종 예상 월 유지비 (Optimized)

| **트래픽 단계** | **예상 DAU** | **월 예상 비용** | **예상 광고/결제 수익 (보수적)** | **이익 (Profit)** |

# Mason's Profit Analysis (손익분기점)

- **Break-even Point (BEP)**: 월 **$45 (약 6만 원)**.
- **수익 모델 검증**:
    - **광고**: 클릭커 게임 특성상 체류 시간이 길어 **노출 수(Impression)**가 압도적입니다. DAU 1만 명이면 하루 광고 노출만 50만 회 이상 가능.
    - **결제**: 경쟁이 과열되면 '커피차($9.99)' 아이템 하루 5개만 팔려도 월 서버비는 전액 충당됩니다.

### 결론

이 프로젝트는 **"Low Risk, High Return"**의 전형입니다.

기술적으로 **Redis Batching(클릭 묶어 보내기)**만 확실하게 구현한다면, 운영 비용은 수익의 1% 미만으로 통제 가능합니다.

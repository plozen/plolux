# 2. 핵심 기술 스택 및 제약 사항 (Tech Stack & Constraints)

아래 규칙은 절대적이며, 어길 시 프로젝트가 붕괴됩니다. 엄격히 준수하십시오.

1. **프로젝트 명세**:
   - Project Name: `Kpop Company League`
   - Directory/Package Name: `kcl` (모든 작업은 이 폴더 내부에서 수행)
2. **프레임워크**: Next.js 14+ (App Router, TypeScript).
3. **패키지 매니저**: **pnpm** (npm, yarn 사용 금지).
4. **스타일링 전략 (중요)**:
   - **SCSS (Sass)** + **CSS Modules** 조합을 사용합니다.
   - **[경고] Tailwind CSS는 절대 사용하지 않습니다.** 프로젝트 생성 시 Tailwind 관련 설정 파일(`tailwind.config.ts`, `postcss.config.js`)이 존재한다면 즉시 제거하고 의존성을 삭제해야 합니다.
5. **아키텍처 패턴**: SCSS 7-1 패턴을 프로젝트 규모에 맞게 최적화한 구조를 사용합니다.
6. **필수 라이브러리**:
   - `classnames`: 조건부 스타일링.
   - `lucide-react`: 아이콘 시스템.
   - `framer-motion`: 고성능 애니메이션.
   - `uuid`: 기기 식별자 생성.
   - `@supabase/supabase-js`: 데이터베이스 연동.
   - `@upstash/redis`: 고성능 캐싱 및 Batching.

# 3. 실행 가이드: 단계별 작업 지시 (Step-by-Step Instructions)

### [Step 1] 프로젝트 초기화 (Initialization)

터미널에서 다음 명령어를 사용하여 Next.js 프로젝트를 생성하십시오. 직접 실행할 수 없는 환경이라면 유저가 실행할 명령어를 정확히 제시하십시오.

npx create-next-app@latest kcl

- -typescript
- -eslint
- -use-pnpm
- -no-tailwind
- -src-dir
- -app
- -import-alias "@/\*"

Note: 설치 옵션에서 Tailwind CSS 사용 여부를 묻는다면 반드시 'No'를 선택해야 합니다.

### [Step 2] 클린업 및 의존성 주입 (Cleanup & Dependencies)

프로젝트 생성 직후, 불필요한 보일러플레이트 코드를 제거하고 필요한 패키지를 설치하십시오.

**파일 삭제**:

- `src/app/globals.css` (SCSS로 대체됨)
- `src/app/page.module.css`

**임시 페이지 수정**:

- `src/app/page.tsx`의 내용을 모두 지우고, 에러 방지를 위해 단순한 `<div>Project Initialized</div>`를 반환하도록 수정하십시오.

패키지 설치:

kcl 디렉토리 루트에서 다음 명령어를 실행하십시오.

pnpm add sass classnames lucide-react framer-motion uuid @supabase/supabase-js @upstash/redis

pnpm add -D @types/uuid

### [Step 3] SCSS 아키텍처 구축 (The 7-1 Pattern)

`src/styles` 디렉토리를 생성하고, 아래의 구조와 파일 내용을 정확히 구현하십시오. 이것은 디자인 시스템의 뼈대입니다.

디렉토리 구조 트리:

src/styles/

├── abstracts/ # 실제 CSS로 컴파일되지 않는 도구들

│ ├── \_variables.scss (색상, 폰트, z-index 변수 - 내용은 비워둘 것)

│ ├── \_mixins.scss (반응형, 레이아웃, 이펙트 믹스인)

│ └── \_functions.scss

├── base/ # 전역 스타일

│ ├── \_reset.scss (강력한 CSS Reset 및 배경 설정)

│ └── \_typography.scss (폰트 패밀리 정의)

├── layout/ # 레이아웃 관련

│ └── \_app-shell.scss (PC/Mobile 하이브리드 레이아웃)

└── main.scss # 진입점 (모든 partial을 import)

**[필수 구현 코드 상세]**

A. src/styles/abstracts/\_mixins.scss

PC 화면에서는 모바일 앱처럼 중앙에 뜨고, 모바일에서는 꽉 차게 보이는 App Shell 로직과 Glassmorphism 효과를 정의합니다.

// 모바일 앱 컨테이너 (PC 대응)

@mixin mobile-container {

width: 100%;

max-width: 480px; // 모바일 최대 너비

margin: 0 auto;

min-height: 100vh;

position: relative;

box-shadow: 0 0 50px rgba(0, 0, 0, 0.5); // PC 화면에서의 깊이감

background-color: #050505; // 앱 배경

overflow-x: hidden;

}

// 유리 질감 효과 (Glassmorphism)

@mixin glass-panel {

background: rgba(255, 255, 255, 0.05);

backdrop-filter: blur(12px);

- webkit-backdrop-filter: blur(12px);

border: 1px solid rgba(255, 255, 255, 0.1);

}

// Flex 중앙 정렬

@mixin flex-center {

display: flex;

justify-content: center;

align-items: center;

}

B. src/styles/base/\_reset.scss

다크 모드를 기본으로 하는 리셋 스타일을 적용합니다.

- {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
  - webkit-tap-highlight-color: transparent; // 모바일 터치 하이라이트 제거
    }

html, body {

width: 100%;

height: 100%;

background-color: #000; // PC 배경 (완전 검정)

color: #fff;

font-family: var(--font-inter), sans-serif;

}

a { text-decoration: none; color: inherit; }

ul, ol { list-style: none; }

button { border: none; background: none; cursor: pointer; font-family: inherit; }

C. src/styles/main.scss

반드시 올바른 순서로 import 해야 합니다.

@import 'abstracts/variables';

@import 'abstracts/mixins';

@import 'base/reset';

@import 'base/typography';

// 추후 layout 및 components 추가 예정

### [Step 4] 환경 설정 및 루트 레이아웃 (Configuration)

A. next.config.mjs 설정

SCSS 파일 내에서 @import 'abstracts/variables'; 처럼 깔끔하게 경로를 사용할 수 있도록 sassOptions를 추가하십시오.

/\*_ @type {import('next').NextConfig} _/

const nextConfig = {

sassOptions: {

includePaths: ['./src/styles'], // 절대 경로 import 지원

},

};

export default nextConfig;

B. src/app/layout.tsx 구현

전역 폰트 설정 및 App Shell 구조를 반영하십시오.

- **폰트**: `next/font/google`의 `Montserrat` (숫자/타이틀용)와 `Inter` (본문용)를 Variable Font 방식으로 로드하십시오.
- **스타일**: 최상단에서 `import '@/styles/main.scss';`를 선언하십시오.
- **구조**: `children`을 `<div className="app-shell">`로 감싸십시오. (이 클래스는 추후 `_app-shell.scss`에서 제어됩니다)

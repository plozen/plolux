# 1. 현재 상황 및 목표 (Context & Goal)

우리는 [Step 1]을 통해 Next.js 14 + SCSS 아키텍처(7-1 Pattern) 기반의 `kcl`프로젝트 초기화를 완료했습니다.
현재`src/styles` 내부에 Mixin과 Reset이 정의되어 있습니다.

당신의 이번 목표는 **"Atomic Design 원칙에 기반한 재사용 가능한 UI 컴포넌트 시스템"**을 구축하고, PC에서도 모바일 앱 경험을 제공하는 **"App Shell 레이아웃"**을 완성하는 것입니다.

---

# 2. 기술적 제약 사항 (Technical Constraints)

1.  **컴포넌트 구조**: `src/components` 디렉토리 하위에 `common`(범용), `layout`(구조), `features`(기능)으로 구분합니다.
2.  **스타일링 규칙**:
    - 모든 컴포넌트는 개별 `*.module.scss`를 가집니다.
    - [Step 1]에서 정의한 `_mixins.scss`와 `_variables.scss`를 적극 활용합니다.
    - `classnames` 라이브러리를 사용하여 조건부 스타일링을 처리합니다 (예: 버튼 variant).
3.  **애니메이션**: 모든 인터랙티브 요소(버튼, 링크 등)는 `framer-motion`을 사용하여 클릭 시 미세한 햅틱 효과(Scale Tap)를 적용해야 합니다.
4.  **디자인 테마**:
    - **Primary Color**: Electric Violet (#8B5CF6)
    - **Secondary Color**: Neon Cyan (#00F0FF)
    - **Background**: Void Black (#050505)
    - **Glassmorphism**: 투명도와 블러 효과가 적용된 UI.
5.  **법적 필수 사항 (Legal)**: 모든 페이지 하단에 반드시 "Unofficial Fan App" 문구가 표시되어야 합니다.

---

# 3. 실행 가이드: 단계별 작업 지시 (Step-by-Step Instructions)

### [Task 1] 전역 변수 정의 (\_variables.scss)

`src/styles/abstracts/_variables.scss` 파일을 열고 K-Pop/Cyberpunk 무드의 컬러 팔레트와 폰트, z-index 시스템을 정의하십시오.

**요구 사항**:

- `$color-primary`, `$color-secondary`, `$color-bg`, `$color-text` 등을 정의.
- `$z-nav`, `$z-modal`, `$z-overlay` 등의 계층 정의.

### [Task 2] App Shell 레이아웃 구현 (Layout System)

PC 화면 중앙에 모바일 앱 크기로 고정되고, 배경에 블러 처리된 이미지가 깔리는 구조를 만드십시오. 법적 리스크 방어를 위한 푸터도 포함합니다.

**파일 생성**:

1.  `src/styles/layout/_app-shell.scss`:
    - `body`에 오버플로우 방지.
    - `.app-shell`: `mixins`의 `mobile-container`를 include하여 중앙 정렬 및 그림자 처리.
2.  `src/components/layout/MobileLayout/index.tsx`:
    - `children`을 받아 렌더링하는 래퍼 컴포넌트.
    - `framer-motion`의 `AnimatePresence`를 적용할 준비.
    - **[중요] Footer 구현**: 컨텐츠 최하단에 작은 글씨(opacity: 0.5)로 "This is an Unofficial Fan App. Not affiliated with any entertainment company." 문구를 고정 노출하십시오.

### [Task 3] 핵심 UI 컴포넌트: Button (Design System)

가장 많이 사용될 버튼 컴포넌트를 구현하십시오.

**파일 위치**: `src/components/common/Button/` (index.tsx, Button.module.scss)
**기능 명세**:

- **Props**: `variant` ('solid' | 'outline' | 'ghost'), `size` ('sm' | 'md' | 'lg'), `isLoading`, `icon`.
- **Interaction**: `whileTap={{ scale: 0.95 }}` 적용.
- **Style**: 'solid'일 경우 그라데이션 배경 적용.

### [Task 4] 하단 네비게이션 바 (Bottom Navigation)

앱의 핵심 네비게이션인 Bottom Tab Bar를 구현하십시오.

**파일 위치**: `src/components/layout/BottomNav/`
**기능 명세**:

- 화면 하단에 `fixed`로 고정.
- `mixins`의 `glass-panel`을 include하여 유리 질감 처리.
- 메뉴: [Home, Ranking, Support, My]. (Map 'Vote' -> 'Support'로 변경)
- `lucide-react` 아이콘 사용.
- 현재 활성화된 탭은 Primary Color로 빛나는 효과(Glow) 적용.

### **Tip: 이 프롬프트의 의도**

1. **변수 통제**: 색상 코드를 헥사코드(#)로 하드코딩하지 않고 `_variables.scss`로 관리하게 하여, 추후 테마 변경(예: 다크/라이트 모드, 시즌별 테마)을 용이하게 합니다.
2. **모바일 퍼스트 강제**: PC 브라우저에서 열어도 모바일 앱처럼 보이도록 강제하는 `App Shell`을 가장 먼저 구현하여 개발 환경을 모바일 기준으로 고정시킵니다.
3. **인터랙션 표준화**: 단순히 모양만 만드는 것이 아니라 `framer-motion`을 기본 탑재하여, 버튼 하나를 눌러도 "고급 앱" 같은 느낌(쫀쫀한 터치감)을 줍니다.

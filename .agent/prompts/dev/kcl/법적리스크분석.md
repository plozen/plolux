# Mason's Legal Risk Assessment (법적 리스크 분석)

## 1. 위험 요소 분석 (The Red Zone)

### A. 상표권 (Trademark) - **위험도: 높음**

- **상황**: 앱 내에서 'HYBE', 'SM' 등의 **공식 로고(CI)**를 그대로 가져와서 사용하는 경우.
- **리스크**: 로고는 각 회사의 등록된 상표입니다. 이를 상업적 용도(광고 수익, 아이템 판매)로 무단 사용하면 상표권 침해로 소송당할 수 있습니다.
- **판례 경향**: 단순히 팬심을 표현하는 비영리 목적이 아니라, **광고/결제 모델이 붙는 순간** '부당이득'으로 간주될 확률이 높습니다.

### B. 부정경쟁방지법 (Unfair Competition) - **위험도: 중간**

- **상황**: 앱의 디자인이나 명칭이 마치 **"공식 앱인 것처럼"** 혼동을 주는 경우.
- **리스크**: "성과 도용 행위"로 간주될 수 있습니다. (남이 쌓은 명성에 무임승차하여 돈을 버는 행위).

### C. 초상권/퍼블리시티권 - **위험도: 최상**

- **상황**: 우승 혜택으로 **아티스트의 사진**을 메인 배너에 거는 경우.
- **리스크**: 소속사가 가장 민감한 부분입니다. 아티스트 사진을 무단으로 상업적 앱의 대문으로 쓰면 즉시 **내용증명(Cease and Desist)** 날아옵니다.

## 2. Mason's Survival Strategy (우회 전략)

법적 시비를 피하면서도 서비스를 운영할 수 있는 **[안전 장치]**입니다.

### 전략 1. "Unofficial" 뱃지 (면책 조항)

- 앱 실행 시 스플래시 화면과 메인 화면 하단에 반드시 다음 문구를 명시해야 합니다.
    
    > "본 서비스는 팬들이 만든 비공식 팬 앱(Unofficial Fan App)이며, 특정 소속사와 관련이 없습니다."
    > 
- 이 문구가 있고 없고의 차이는 법적 방어에서 매우 큽니다.

### 전략 2. 로고 대신 "텍스트" 또는 "팬아트" 사용

- **공식 로고(CI) 원본 사용 지양**:
    - 대신, **세련된 폰트로 회사 이름만 표기**하거나,
    - 저작권이 없는 **미니멀한 아이콘/심볼**로 대체하는 것이 가장 안전합니다. (예: JYP 대신 고릴라 아이콘, SM 대신 분홍색 곰돌이 등 팬들만 아는 은유적 아이콘 사용 → 오히려 퀴즈 같아서 재미 요소가 됨).

### 전략 3. 이미지는 "유저 생성 콘텐츠(UGC)"로 돌리기

- **전략**: 개발자가 직접 아티스트 사진을 올리지 마세요.
- **구현**: 우승 혜택인 '메인 배너' 영역을 팬들이 직접 찍은 사진(직찍)이나 직접 그린 팬아트를 **업로드**하게 만드세요.
- **방어 논리**: "우리는 플랫폼만 제공했고, 사진은 유저가 올린 것이다" (DMCA Safe Harbor와 유사한 논리). 문제 시 해당 사진만 삭제하면 앱 전체는 보호됩니다.

### 전략 4. Takedown Policy (즉시 삭제 정책)

- 소속사에서 연락이 오면, **"즉시 해당 소속사 데이터를 내리거나 아이콘을 수정하겠다"**는 태도를 취해야 합니다.
- 보통 소속사는 바로 소송하지 않고 **"사용 중지 요청"**을 먼저 보냅니다. 이때 바로 수정하면 배상 책임까지 가는 경우는 드뭅니다.

## 3. 요약 및 실행 가이드

1. **회사 이름(텍스트)**: 사용 가능 (식별을 위한 명칭 사용은 허용됨).
2. **공식 로고(이미지)**: **사용 금지 권장**. (비슷한 느낌의 텍스트 로고로 직접 디자인해서 쓰세요).
3. **아티스트 사진**: **절대 사용 금지**. (유저가 직접 올리게 유도하세요).
4. **수익화**: "비공식 팬 앱"임을 명시하면 광고 수익 정도는 묵인되는 경우가 많으나, 대놓고 '공식'인 척하면 위험합니다.

결론:

앱 이름이나 디자인에서 "팬이 만든 놀이터" 느낌을 강조하면 괜찮습니다. 너무 고퀄리티로 "공식 앱"처럼 만들지만 않으면 됩니다.
